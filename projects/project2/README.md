## 2. 🎯 목표 서비스 구현 및 실제 구현 정도

- 사용자 맞춤형 도서 추천 웹 서비스 구현

✅ **기능 완성도**: 처음 목표했던 기능은 대부분 구현 완료

⏱ **개발 어려움**: 버그 수정 및 기능 추가로 예상보다 시간 소요 많음

## 3. 🗂 데이터베이스 모델링 (ERD)

### ▶ 관계 설명 요약

- **User**:
  - 여러 개의 북마크 폴더, 게시글, 댓글, 방명록 작성/수신 가능
  - 사용자 간 팔로우 관계 가능
  - 선호 카테고리 설정 가능
- **Book**:
  - 하나의 카테고리에 속함
  - 여러 게시글, 북마크, 추천 관계를 가질 수 있음
- **Post**:
  - 여러 댓글 및 좋아요를 가짐
- **Bookmark**:
  - 북마크 폴더는 여러 북마크를 포함

## 4. 🧠 도서 추천 알고리즘

- **저자 기반 추천**: 사용자가 선호하는 저자의 도서를 추천
- **장르 기반 추천**: 선호 장르 기반 도서 추천
- **랜덤 추천**: 새로운 도서를 위한 무작위 추천

## 6. 🤖 생성형 AI 및 외부 API 활용

- **gTTS 음성 변환**: 도서 설명을 음성으로 재생
- **Wiki API**: 저자 배경 정보 제공

## 7. 📝 느낀 점

- 처음 목표한 기능은 대부분 구현되었지만 **UI를 완성하는 데 충분한 시간이 부족**했다.
- 밤을 새며 버그를 해결했지만, 기능을 추가할 때마다 **예상치 못한 다른 버그가 계속 발생**했다.
- 그럼에도 완성된 결과물을 보며 **큰 보람을 느꼈고**, 각종 오류 상황에서도 **협업하며 해결하는 과정에서 팀워크와 개발 능력이 많이 향상**되었다.
- 여러 참고 사이트를 오가며 공부하고 적용하는 시간이 의미 있었고, **서로 파트를 바꿔가며 협업한 경험**은 좋은 배움이었다.
